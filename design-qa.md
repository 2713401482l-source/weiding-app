# 《白境》移动端兼容与商用视觉 QA

## 本轮目标

- 优先保证手机端在小屏、常规全面屏、横屏与平板上的内容可达性。
- 适配 iOS Safari、Android Chrome 及常见 WebView 的安全区、动态地址栏和软键盘行为。
- 在不破坏“现代美术馆克制感 / Typographic Silence”的前提下，提高加载、动画、触控与焚烧效果的稳定性。

## 已落地的兼容与性能策略

- 使用 `100dvh` 与 `100vh` 双层回退，并通过 `env(safe-area-inset-*)` 处理刘海、圆角和底部手势区。
- 使用 `VisualViewport` 识别软键盘和可视区域变化；键盘弹出时隐藏底部 TabBar，避免遮挡输入与确认按钮。
- 320px 短屏允许状态页纵向滚动；手机横屏采用双栏紧凑布局，全部状态仍可访问。
- 主页面横滑保留系统左右边缘手势区，并排除按钮、输入框、画布和滑杆，减少误触。
- 移除外部网络字体依赖，改用系统中文字体栈；降低首屏阻塞和弱网字体闪烁。
- 手写画布限制设备像素比，并根据低性能设备降低粒子数量和刷新率；尺寸变化时保留已写笔迹。
- 对省流量、低内存或低核心数设备启用低功耗效果；支持减少动态与减少透明度偏好。
- Service Worker 不拦截音频 Range 请求，导航采用网络优先，避免音频元数据、拖动和离线缓存冲突。
- 音频增加加载、失败、重试与禁用状态；全屏 API、媒体查询监听和随机 ID 均提供旧版 Safari 回退。

## 三轮自主验收

### 第一轮：多尺寸视觉与布局

- 320×568：短屏状态页可滚动，TabBar 不遮挡底部状态。
- 390×844、430×932：主要手机视口布局稳定，安全边距与底部手势区生效。
- 844×390：横屏双栏结构可用，内容无横向溢出。
- 768×1024：平板仍保持聚焦的手机画幅，不产生过宽、过散的内容。
- 浅色、深色、减少动态、强制颜色模式均有兼容规则。

### 第二轮：关键交互与防误触

- 状态选择只改变当前选项，不会松手自动进入；点击代表场景后进入冥想。
- 播放器默认收起，展开后具备后退 15 秒、播放/暂停、前进 15 秒和提前结束；入口始终位于面板下方。
- 阅后即焚输入超过 2 秒仍完整保留，只有点击“让它消散”才执行焚烧并清空。
- 反馈选择后短暂显示温和建议，并约 1.5 秒后返回状态调整首页。
- 四个常驻 Tab 顺序为：状态调整、阅后即焚、时间流转、设置。

### 第三轮：自动化、性能与安全

- 单元测试：2/2 通过。
- 生产构建：通过。
- 依赖审计：0 个已知漏洞。
- Lighthouse 手机模拟（390×844、默认移动网络/CPU 节流）：Performance 94、Accessibility 100、Best Practices 100。
- 核心指标：FCP 2.4s、LCP 2.4s、TBT 0ms、CLS 0、Speed Index 2.5s。
- 320×568 与横屏无节流视觉基线：Performance 100；各主要尺寸无障碍与最佳实践均为 100。

## 仍需实体设备抽检的边界

Windows 自动化无法真正模拟 iPhone Safari 的系统触觉、静音策略、全屏限制和扬声器音色。代码已提供兼容降级，但上线前仍建议至少用一台 iPhone 与一台 Android 实机完成：六段音频试听、软键盘输入、横竖屏旋转、添加到主屏幕和弱网恢复测试。

## 本轮参考基线

- WebKit：iPhone 安全区与 `viewport-fit=cover`。
- MDN：`VisualViewport`、安全区环境变量、减少动态偏好。
- Chrome Developers：移动端软键盘引起的视口变化。
- web.dev：INP、CLS、响应式布局和字体性能建议。

最终结果：自动化验收通过，无已知 P0/P1 阻断项。
